## 一、产品概述

**沈航智学 AeroTutor** 是一款 NotebookLM 风格的多知识库自适应学习 Agent，专为航空航天专业课程学习场景打造。系统以"知识空间"为核心组织单元，融合 RAG 检索增强生成、LLM 智能工具调用、认知诊断分析等技术，为学生提供从知识入库、检索问答、测验自评到薄弱诊断的全链路自适应学习体验。

- **公开访问**：https://sau-aerotutor-9a5dscagkznibfd2ppsaog.streamlit.app
- **技术栈**：Streamlit + LangChain + FAISS + DeepSeek/GLM
- **部署方式**：Streamlit Cloud（零运维，即开即用）

### 2.2 技术选型说明

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| UI 框架 | Streamlit 1.31+ | 纯 Python 构建，零前端代码，开发效率高 |

| 文件解析 | PyMuPDF + python-pptx | PDF/PPTX/TXT/MD 全格式覆盖 |
| 可视化 | Plotly 5.22+ | 知识图谱交互渲染预留 |

## 三、核心功能

### 3.1 多知识空间管理（NotebookLM 风格）

- **独立向量索引**：每个知识空间对应一个独立的 FAISS 索引文件，向量完全不互通
- **分类体系**：课程课件 / 学习笔记 / 论文文献 / 参考资料 / 其他
- **文件解析**：上传 PDF 课件、PPT 讲稿、TXT/Markdown 笔记自动分块向量化
- **一键切换**：勾选侧边栏知识空间即可激活，支持多选联合检索

### 3.2 RAG 检索增强生成

- **多库联合检索**：根据用户勾选的知识空间，跨索引并行检索
- **答案溯源**：每个回答自动标注引用文献（文件名 + 知识空间名）
- **语义分块**：500 字滑动窗口切分，中文句号/分号优先的递归分割策略

### 3.3 智能学习工具箱（6 Tools）

| 工具 | 触发方式 | 功能 |
|------|---------|------|
| 🔍 知识检索 | 自然提问 | 跨知识库检索并生成带引用的回答 |
| 📝 出题测验 | "出 5 道题" | 基于知识库自动生成选择题 + 简答题 |
| 🃏 知识闪卡 | "制作闪卡" | 提取核心概念生成正面/反面闪卡 |
| 📊 认知诊断 | "分析薄弱点" | 基于对话历史诊断知识掌握情况 |
| ⚡ 速查摘要 | "总结重点" | 一键生成核心概念 + 必会公式 + 常见误区 |
| 🎙️ 知识播客 | "生成播客" | 师生对话式知识讲解脚本（王教授 + 小明） |

### 3.4 AI 自动批改

- 选择题即时判对错，标注正确选项 + 解析
- 简答题调用 AI 按要点打分（满分 20），逐条点评
- 得分 < 60% 自动收录错题本

### 3.5 认知诊断 + 学习航线

- **三阶诊断模型**：已掌握（mastered）→ 薄弱（weak）→ 未覆盖（unknown）
- **个性化航线**：基于诊断结果生成 3-5 步学习路径，标注每步时长和资源类型
- **航空特色**：用飞行 check list、飞行计划等比喻组织学习内容

### 3.6 复习智库

- **错题本**：自动收录 + 手动标记已复习，追踪复习次数
- **闪卡集**：CSS 3D 翻转动画，正面概念 → 反面详解
- **诊断报告**：历史诊断归档，追踪进步轨迹
- **学习航线**：步骤化学习路线指引
- **学习统计**：连续复习天数、测验次数、闪卡复习次数

### 3.7 对话管理

- **文件夹分组**：拖拽对话到文件夹，支持嵌套子文件夹
- **持久化存储**：本地 JSON 文件，Streamlit Cloud 自动恢复
- **多轮记忆**：保留最近 20 轮对话上下文

### 3.8 多 API 供应商热切换

- **预设供应商**：DeepSeek / 智谱 GLM / 自定义 OpenAI 兼容接口
- **配置持久化**：设置保存到 JSON 文件，重启不丢失
- **连接测试**：内置 API 连通性检测，一键验证 Key 有效性
- **即时生效**：切换供应商无需重启应用

## 四、使用场景

### 场景一：课后自主复习

> 小明刚上完《空气动力学》课程，将老师的 PPT 上传到「空气动力学」知识空间。他输入"用伯努利原理解释机翼升力"，AeroTutor 从课件中检索相关内容，给出带引用的详细回答。随后他点击「生成测验」，系统出了 3 道题（1 道选择 + 2 道简答），AI 自动批改并将错题收录错题本。

### 场景二：考前系统复习

> 期末临近，小红勾选了「飞行原理」「航空发动机」「材料力学」三个知识空间，点击「诊断薄弱点」。系统分析她本学期的对话记录，标记出 5 个薄弱点（如"涡轮叶片冷却"）和 3 个未覆盖知识盲区，自动生成了一条 5 步学习航线：从基础的"热力学循环"到进阶的"气膜冷却技术"，每步标注建议学习时长和推荐资源。

### 场景三：碎片时间闪卡复习

> 在等校车时，小刚打开 AeroTutor 点击「制作闪卡」，系统基于知识库生成了 5 张闪卡（正面"雷诺数"，反面详细解释和相关公式）。他进入复习智库的闪卡集，点击卡片即可翻转查看答案，利用碎片时间巩固知识点。

### 场景四：知识播客沉浸学习

> 小李对《飞行力学》中的"静稳定性"概念一知半解，他让 AeroTutor 生成知识播客。系统生成了王教授和小明的 12 轮师生对话脚本，用生动比喻（"就像不倒翁，重心在压力中心下方就自动回正"）把抽象概念讲透。

### 场景五：多供应商灵活切换

> 张同学有 DeepSeek 的免费额度，李同学有 GLM 的学术赠金。两人各自在 AeroTutor 的设置面板中输入自己的 API Key，切换到对应供应商，互不影响。

## 五、创新点

### 5.1 NotebookLM 风格多知识库隔离

借鉴 Google NotebookLM 的设计理念，每个学科/章节对应一个独立的知识空间，向量索引完全隔离。学生上传不同课程的课件后，系统确保检索时不会跨学科"串味"，答案溯源精确到文件名。

### 5.2 意图预检测 + 工具直通架构

传统 Agent 依赖 LLM 判断是否调用工具，存在延迟高、判断不稳定问题。AeroTutor 在 Agent 层之前加入**中英文关键词意图预检测**，对"出题""闪卡""诊断""速查""播客"五类高频操作直接命中执行，绕过 LLM 推理环节，响应速度提升 3-5 倍，准确率 100%。

### 5.3 "出题 → 批改 → 错题本 → 诊断 → 航线"五步闭环

系统不是孤立的工具集合，而是形成完整的自适应学习闭环：

```
知识入库 → 检索学习 → 出题自测 → AI 批改 → 错题收录
                                              ↓
学习航线 ← 认知诊断 ← 薄弱分析 ← 测验统计 ← 错题回顾
```

每一步的输出都是下一步的输入，形成螺旋上升的学习飞轮。

### 5.4 航空航天领域定制

- **播客角色设定**：王教授（航空专家）+ 小明（飞行器设计专业大二学生）
- **回答风格**：用飞行 check list、飞行计划、座舱仪表等航空比喻讲解概念
- **知识空间分类**：课程课件 / 论文文献 / 参考资料，贴合工科学习场景

### 5.5 零成本私有部署能力

系统支持学生自备 API Key，平台本身零推理成本。FAISS 本地向量检索无需外部数据库，sentence-transformers 本地 Embedding 无需付费 API。小学期/社团可直接部署使用，无需预算审批。

### 5.6 多供应商热切换

区别于市面上"绑定单一模型"的 AI 学习工具，AeroTutor 内置 DeepSeek、GLM、自定义三档供应商，学生可根据手头的 API 额度自由切换。设置面板包含连接测试功能，一键验证配置有效性。

### 5.7 Gemini 风格极简交互

- 悬浮侧边栏（收起 56px → 展开 320px）
- 对话气泡圆角 + 渐变品牌色标题
- 打字机流式输出
- CSS 3D 翻转闪卡
- 文件夹拖拽排序
